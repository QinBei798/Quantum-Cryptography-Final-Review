# 核心摘要

> 本章为【量子两方不信任博弈】专项。核心考点高度集中于 **[[比特承诺]]与[[量子掷币]]的阶段划分、隐蔽性与约束性的博弈定义**。必须死记硬背的理论死线是 **[[Mayers-Lo-Chau定理|BC不可能性定理]]**（完美量子比特承诺在物理上绝对不存在）。协议部分重点考查 **[[Ambainis掷币协议]]** 的三维空间基底结构及偏置限制（$\text{偏}\le 0.354$）。

# 第8章 量子比特承诺与量子掷币

## 8.1 量子比特承诺 (Bit Commitment, BC)

### 8.1.1 概念直觉（保险箱比喻）
- Alice 是一名股票经纪人，她预测了一个比特 $b$。她把 $b$ 写在纸上放进保险箱，并把保险箱留在 Bob 处，自己拿走钥匙。一个月后，Alice 带着钥匙来打开保险箱，向 Bob 证实她当年的预测。

### 8.1.2 协议的两大阶段与两大核心特性（期末必考名词解释）
1. **承诺阶段 (Commitment phase)**：Alice 选定一个比特 $b$，并向 Bob 发送一段包含 $b$ 信息的量子/经典流。
   - **[[隐蔽性|隐蔽性 (Concealing)]]**：在打开阶段前，Bob 无论使用何种物理测量手段，都**无法获知** $b$ 的任何信息。
2. **打开阶段 (Opening phase)**：Alice 公开证据（如密钥或辅助参数），Bob 进行物理测量验证。
   - **[[约束性|约束性 (Binding)]]**：Alice **无法欺骗/篡改**。如果她最初承诺的是 $0$，她绝不可能通过伪造在打开阶段成功骗 Bob 说是 $1$。

---

### 8.1.3 理论重磅炸弹：[[Mayers-Lo-Chau定理|量子BC不可能性定理 (BC Impossibility)]]
> ❌ **期末极高频辨析误区**：“基于量子力学的完美隐蔽且完美的约束的比特承诺协议是存在的。” **—— 错误！**

- **MLC 定理定理内容**：1997年由 Mayers、Lo(卢振球)、Chau 严格证明：**如果一个量子比特承诺协议具有完美的隐蔽性，那么它绝对不具有完美的约束性**。也就是说，无条件安全的完美非相对论量子比特承诺协议在物理上**不存在**。
- **作弊的物理本质：局域纠缠变换**
  - 如果协议完美隐蔽，意味着无论 $b=0$ 还是 $b=1$，Bob 端收到的系统密度矩阵都完全相同（$\rho_0 = \rho_1$）。
  - 根据 Hughston-Jozsa-Wootters (HJW) 定理，Alice 在自己一端必然可以将承诺比特与一个大辅助系统制备成一种**全局的大纠缠态**。
  - 在打开阶段前，Alice 根本不需要做决定。等到要打开时，如果她想说 $0$，就对她手里的粒子施加局域么正变换 $U_0$ 后测量；如果想说 $1$，就施加局域么正变换 $U_1$ 后测量。**由于爱因斯坦非定域性，Alice 可以在完全不惊动 Bob 的情况下，单方面任意篡改承诺值。**



- **现实妥协出路**：放弃无条件安全假设。通过引入**量子存储时间有限长（含噪存储模型）**、相对论时空隔离，或者使用 **[[Ambainis掷币协议|三维基底不完美方案]]**，使欺骗概率（偏）降到可容忍的范围内。

## 8.2 量子掷币 (Quantum Coin Flipping)

### 8.2.1 概念与博弈核心
- 两个互不信任的远端参与者 Alice 和 Bob，希望通过网络达成一个**完全公平的随机比特 $c \in \{0, 1\}$**（就像抛硬币决定谁洗碗）。
- **游戏规则**：
  - 如果双方都诚实，输出 $0$ 和 $1$ 的概率必须严格为 $50\%$对 $50\%$。
  - 如果一方不诚实（想强行控制结果为自己想要的 $a$），另一方诚实，则不诚实方获胜的概率上限定义为 $P_{\max}$。
- **[[偏置|偏置 (Bias)]]的量化指标**：
  $$\epsilon = P_{\max} - \frac{1}{2}$$
  密码学要求：偏置 $\epsilon$ 必须越接近 0 越好。若 $\epsilon = 0.5$，说明协议完全崩溃，坏人可以 100% 控制比赛。

---

### 8.2.2 AMBAINIS 掷币协议（核心大题分析）

Ambainis 协议是两方量子掷币的里程碑方案，它巧妙利用了**三维 Hilbert 空间**中的基底正交折衷。

#### 1. 物理状态制备
Alice 随机选择基 $\mathcal{B}_a'$，其中 $a=0, 1$ 代表不同的基底体系。空间是三维的，基底中包含两个正交分量以及一个独立分量：
- **当 $a=0$ 时**：基底为 $\mathcal{B}_0' = \{|\phi_{0,0}\rangle, |\phi_{0,1}\rangle, |2\rangle\}$
  - $|\phi_{0,0}\rangle = \alpha|0\rangle + \beta|1\rangle$
  - $|\phi_{0,1}\rangle = \beta|0\rangle - \alpha|1\rangle$
- **当 $a=1$ 时**：基底为 $\mathcal{B}_1' = \{|\phi_{1,0}\rangle, |\phi_{1,1}\rangle, |2\rangle\}$
  - $|\phi_{1,0}\rangle = \alpha|0\rangle - \beta|1\rangle$
  - $|\phi_{1,1}\rangle = \beta|0\rangle + \alpha|1\rangle$
*(其中系数选取常为 $\alpha = \cos(\pi/8)$, $\beta = \sin(\pi/8)$)*

#### 2. 交互博弈步骤（简答必背）
1. **Alice 承诺**：Alice 随机选择一个硬币值 $a \in \{0, 1\}$（作为她的掷币承诺），并随机选 $x \in \{0,1\}$，制备出状态 $|\phi_{a,x}\rangle$ 发给 Bob。
2. **Bob 挑战**：Bob 在不知道 $a$ 的情况下，随机选择一个经典比特 $b \in \{0, 1\}$ 发给 Alice。
3. **Alice 打开**：Alice 收到 $b$ 后，向 Bob **公布**她最初选的 $a$ 和 $x$。
4. **Bob 验证**：Bob 使用 Alice 宣布的 $a$ 所对应的基 $\mathcal{B}_a'$ 去测量他留存的粒子。
   - 如果测量出来的状态结果与 Alice 宣称的 $x$ **不匹配/不一致**，Bob 立刻断定 Alice 实施了**欺骗**，中止协议。
   - 如果匹配通过，则本次网络掷币的最终合并结果为：**$c = a \oplus b$**。



#### 3. 安全性量化结论
- **隐蔽性保障**：由于不管 $a=0$ 还是 $1$，其总密度矩阵在三维空间中存在极大的空间重叠（$\rho_{a=0} \neq \rho_{a=1}$ 但两者的迹距离很小），Bob 在第二步强行去测根本无法 100% 猜出 $a$。
- **约束性保障**：如果 Alice 在第四步想篡改 $a$ 来控制结果，由于不同基底之间的非正交投影重叠，她被 Bob 测出冲突进而抓获的概率极高。
- **偏置天花板**：经过严格优化，Ambainis 协议被证明其**最大偏置 $\epsilon = \frac{1}{\sqrt{2}} - \frac{1}{2} \approx 0.354$**。不诚实方最高只能将赢面控制在 $85.4\%$ 左右，无法达到 100%。

# 待查证（期末突击疑点）

- [ ] **Kitaev 掷币可达偏置的理论底线**：课件中给出了经典 Ambainis 协议的偏置为 $0.354$。然而根据 Kitaev 经典定理证明，两方弱量子掷币（Weak Coin Flipping）的偏置可以无限趋近于 0，而强量子掷币的偏置硬物理死线是 $\epsilon \ge \frac{1}{\sqrt{2}} - \frac{1}{2}$。在期末考卷的大型论述题中，如果要求“论证两方强/弱掷币在物理可实现度上的本质极限差异”，作答时关于“强/弱”定义（弱掷币中双方目标输赢利益不对称，如一方只想出0，另一方只想出1）的严谨文字表述界限是什么？
- [ ] **Ambainis 协议中 Bob 测错状态时的欺骗概率计算大题**：如果 Alice 初始发射了 $|\phi_{0,0}\rangle$，但在第三步因为看到 Bob 发了不符合她心意的 $b$，铤而走险强行宣布 $a=1, x=0$。此时 Bob 会换用 $\mathcal{B}_1'$ 基去投影测量原本的 $|\phi_{0,0}\rangle$。要求在草稿纸上熟练写出内积模平方的计算：$P = |\langle \phi_{1,0}|\phi_{0,0}\rangle|^2 = |(\alpha\langle 0| - \beta\langle 1|)(\alpha|0\rangle + \beta|1\rangle)|^2 = |\alpha^2 - \beta^2|^2 = |\cos^2(\pi/8) - \sin^2(\pi/8)|^2 = \cos^2(\pi/4) = 0.5$。即 Alice 的欺骗有 50% 概率直接暴露。考前需对这个代数推导流程极其熟悉。
