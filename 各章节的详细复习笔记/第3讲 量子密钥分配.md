# 第3章 量子密钥分配 (QKD) 核心协议

## 3.1 BB84 协议的数学细节与后处理（重点大题！）

### 3.1.1 四种极化状态
BB84 使用两组**互补（非正交）**的基，共 4 个状态：
- **$Z$ 基（计算基 / 十字基）**：$\{|0\rangle, |1\rangle\}$，分别代表经典比特 `0` 和 `1`。
- **$X$ 基（对角基 / 叉字基）**：$\{|+\rangle, |-\rangle\}$，分别代表经典比特 `0` 和 `1`。
  - $|+\rangle = \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)$
  - $|-\rangle = \frac{1}{\sqrt{2}}(|0\rangle - |1\rangle)$

### 3.1.2 完整执行步骤（填空/简答必考步骤顺序）
1. **制备与发送**：Alice 随机生成 $n$ 位经典比特串，并随机为每个比特选择 $Z$ 基或 $X$ 基，制备对应的单光子状态通过量子信道发给 Bob。
2. **随机测量**：Bob 收到每个光子时，在不知道 Alice 所选基的情况下，随机选择 $Z$ 基或 $X$ 基进行测量，记录测量结果。
3. **基比对 (Sifting)**：双方通过**不安全的经典信道**公开各自使用的测量基。
   - 留存：保留双方`测量基相同`的轮次，这部分比特称为 **原始密钥（生密钥 / Raw Key）**。理论上过滤掉 50% 的轮次。
4. **后处理 (Post-processing)**：
   - **[[窃听检测]]**：随机抽取生密钥中的一部分比特公开比对。如果错误率 $e$ 高于某安全阈值（理论上限约为 $11\%$），说明存在窃听，立即放弃整批密钥。
   - **[[纠错|原始纠错 (Error Correction)]]**：利用经典编码（如 Cascade 协议或 LDPC 码）消除因信道噪声或微弱窃听带来的比特差异。
   - **[[保密增强 (Privacy Amplification)]]**：使用单向 Hash 函数（或压缩矩阵）对纠错后的密钥进行线性压缩，消除 Eve 掌握的任何部分残余信息，提纯出最终的绝对安全密钥。

### 3.1.3 截获-重发攻击 (Intercept-Resend) 25% 错误率的严格推导
> **考试经典大题**：证明 Eve 实施截获-重发攻击会在 Alice 和 Bob 的生密钥中引入 25% 的固有错误率。



**证明步骤**：
1. 考虑 Alice 成功保留在生密钥中的某一个比特（即 Alice 和 Bob 在这轮随机选了**相同的基**，假设都选了 $Z$ 基，Alice 发送的是 $|0\rangle$）。
2. Eve 在信道中间拦截，由于她不知道基，她有 **$50\%$ 的概率选对基**（$Z$ 基），有 **$50\%$ 的概率选错基**（$X$ 基）。
   - **情形 A（Eve选对基，概率 1/2）**：Eve 用 $Z$ 基测 $|0\rangle$，100% 测得结果 0，状态不改变，仍为 $|0\rangle$。Eve 伪造一个 $|0\rangle$ 发给 Bob。Bob 用 $Z$ 基测，100% 得到 0。此时 **错误率为 0**。
   - **情形 B（Eve选错基，概率 1/2）**：Eve 用 $X$ 基测 $|0\rangle$。根据测量原理，$|0\rangle = \frac{1}{\sqrt{2}}(|+\rangle + |-\rangle)$，Eve 有 $50\%$ 概率测出 $|+\rangle$，50% 概率测出 $|-\rangle$。无论测出哪一个，状态都会**塌缩**。假设 Eve 测出 $|+\rangle$，她就重新制备一个 $|+\rangle$ 发给 Bob。
3. 此时 Bob 收到 $|+\rangle$，他使用 $Z$ 基测量。由于 $|+\rangle = \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)$，Bob **测得结果为 `0` 或 `1` 的概率各占 $50\%$**。
4. **总错误率计算**：
   $$\text{Error Rate} = P(\text{情形 A}) \times 0 + P(\text{情形 B}) \times P(\text{Bob 在错态下测出比特 1})$$
   $$\text{Error Rate} = \frac{1}{2} \times 0 + \frac{1}{2} \times \frac{1}{2} = \frac{1}{4} = 25\%$$
   **结论**：Eve 的截获-重发攻击不可避免地会引入 $25\%$ 的高错误率，这远超系统正常的噪声阈值，必然会被安全检查发现。

## 3.2 B92 协议：两态非正交

- **核心精简**：Alice 只用两个**非正交**的状态：
  - 比特 `0` $\rightarrow$ $|0\rangle$ ($Z$ 基)
  - 比特 `1` $\rightarrow$ $|+\rangle$ ($X$ 基)
- **Bob 的测量策略**：Bob 随机选择用 $X$ 基的 $|-\rangle$ 或 $Z$ 基的 $|1\rangle$ 进行投影测量。
- **筛选逻辑**：
  - 如果 Alice 发 $|0\rangle$，Bob 用 $|1\rangle$ 测，内积为 0，绝对测不到结果。
  - 如果 Alice 发 $|0\rangle$，Bob 用 $|-\rangle$ 测，因为 $\langle -|0\rangle = \frac{1}{\sqrt{2}}$，有 50% 概率能坍缩测出结果。
  - **结论**：Bob 只有在**测到确定性清除状态**（如用 $|1\rangle$ 测出了结果，说明 Alice 绝对不可能发过 $|1\rangle$ 相关的基，反推 Alice 必发了 $|+\rangle$，对应比特 `1`）时，该轮才是有效位。
- **优缺点**：结构比 BB84 更精简，但效率较低，生密钥落地成码率只有 $25\%$。

## 3.3 E91 协议：基于量子纠缠

- **核心机制**：Alice 和 Bob 之间没有直接发送单光子，而是由一个可信中继源（或其中一方）不断向两端发射最大纠缠对（**[[Bell态]]**，如 $|\Psi^+\rangle = \frac{1}{\sqrt{2}}(|01\rangle+|10\rangle)$）。
- **测量基选择**：双方各自独立、随机地从**三种不同的极化方向（夹角互成对应比例）**中选基测量。
- **双重功能划分**：
  - **生成密钥**：当 Alice 和 Bob 偶然选了**完全相同的方向基**时，由于纠缠的强关联性，两边的测量结果会完美呈现 100% 的相反（或相同）对应。取反后即可直接安全作为密钥。
  - **防范窃听**：当双方选了**不同的方向基**时，将这些数据收集起来去公开验证 **[[Bell不等式|Bell 不等式（CHSH 不等式）]]**。
- **防偷看原理**：如果 Eve 在中继到两端的信道间企图偷看或偷换粒子，就会破坏粒子对之间的量子纠缠度。一旦纠缠被破坏，CHSH 值的测量计算结果就会退化跌回经典物理的上限值 $\le 2$。只有检测到 **CHSH 值大于 2**（最大可达 $2\sqrt{2}$），才能肉眼确证该信道绝对没有被任何人动过。

## 3.4 正交态 QKD 协议（GV95、LL02）与信息分割

> ❌ **易错概念辨析**：常人误以为“量子密码必须使用非正交态，正交态协议不能保密”。这是错误的。

### 3.4.1 [[信息分割]] 的物理本质
- **GV95、LL02 协议的原理**：Alice 和 Bob 之间传输的状态在完整的空间里是正交的。但是，Alice 将单粒子分成了**两个在时间/空间上分立的波包**（或者分步发送双粒子纠缠对中的两个粒子），在信道传输期间，**任意单次只传输信息的一部分**。
- **Eve 的困境**：在任何一个特定传输时间点上，Eve 在信道中能够截获截取的，本质上只是`局部的信息片段`。
- 从数学密度矩阵来看，这个局部片段在局部呈现出来的状态依然是**最大混合态或彼此非正交态**。Eve 强行去测，必然因为拿不到完整的整体而引入干扰塌缩，从而被后处理发现。
- **期末简答标准结论**：正交态 QKD 协议和非正交态协议并不对立，它们的本质物理防御精髓都是 **[[信息分割]]（Information Splitting）**。
