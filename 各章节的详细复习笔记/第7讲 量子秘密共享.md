# 第7章 量子秘密共享 (QSS)

## 7.1 量子秘密共享的基本概念

### 7.1.1 什么是秘密共享？
- **`(k, n)` 门限方案 (Threshold Scheme)**：分发者（拥有者）把消息加密成 $n$ 份，分别发给 $n$ 个接收者（代理人）。要求他们当中的**任意 $k$ 个人**联合可以重构出这条消息，而任何人数**少于 $k$ 的组合**都得不到这条消息的任何信息。
- **最常见模型**：QSS 中最常见的是 **`(n, n)` 门限方案**（所有代理人必须全部合作才能解密）。
- **基本假设防线**：至少有一个代理人是**诚实**的。
- **共享分类**：根据共享信息的不同，分为共享**经典信息**和共享**量子信息**两类。

## 7.2 HBB 协议：基于三粒子 GHZ 态的 QSS（绝对核心考点）

HBB 协议（1999年提出）是第一个利用三粒子最大纠缠态（GHZ 态）实现的 `(2, 2)` 门限量子秘密共享协议。

### 7.2.1 物理载体：GHZ 态
系统共享一组三粒子纠缠态：
$$|\psi\rangle_{ABC} = \frac{1}{\sqrt{2}}(|000\rangle + |111\rangle)_{ABC}$$
下标 $A, B, C$ 分别代表属于分发者 Alice、代理人 Bob 和 Charlie 的粒子。

### 7.2.2 协议执行与解密推导（大题必考）
1. **分发与选择**：Alice、Bob 和 Charlie 分别随机选择 **$X$ 基** 或 **$Y$ 基** 来测量自己手里的粒子。
2. **公式演化直觉**（考前必须看一眼这个基底映射关联）：
   - 如果三人都选 $X$ 基，状态在 $X$ 基下展开，其测得的特征值（或01比特）满足强关联。
   - 物理限制定理：三人分别测得结果 $a, b, c \in \{0, 1\}$，由于纠缠限制，他们的结果在数学上存在一种**互补异或关系**。
3. **协同重构密钥**：
   - Alice 要将一个秘密比特传给 Bob 和 Charlie 共享。
   - **关键点**：Bob 一个人去测，他的结果完全是 50:50 随机的混合态，拿不到任何线索；Charlie 也是如此。
   - **重构过程**：Bob 和 Charlie 必须通过经典信道相互告知对方自己的测量基和结果。**当且仅当两人合作时**，他们可以联合算出 $b \oplus c$。
   - 随后，他们与 Alice 公开的测量基进行比对。如果基匹配，Bob 和 Charlie 就可以利用 $b \oplus c$ 的结果完美推导反算出 Alice 当时那一轮的测量结果 $a$。
   - 最终，Alice 就可以用这个只有“Bob+Charlie联合体”才知道的序列 $a$ 作为单次垫片，来实现对真实秘密的无条件安全共享。

## 3.3 更多 QSS 协议架构

### 1. 基于单光子的 QSS 协议
- **物理机制**：不需要制备复杂的多粒子纠缠源。Alice 连续发送单光子序列（随机调制在 $Z$ 基或 $X$ 基）。
- **流转路径**：光子先发给第一个代理人 Bob，Bob 对光子随机施加局域么正变换（如 $I$ 或 $i\sigma_y$），再转发给第二个代理人 Charlie。Charlie 同样施加局域变换后，最后做测量或发回给 Alice 测量。
- **安全性**：利用**块传输 (Block Transmission)** 技术进行侧信道与窃听检测，同样能达到 `(2, 2)` 门限效果。

### 2. 基于纠缠交换 (ES) 的多方 QSS 协议（ZM协议）
- **原理**：利用两对双粒子纠缠态（如两对 Bell 态），通过在一端执行纠缠交换测量（Bell 基测量），强行让原本没有直接交集的两个远端粒子瞬间建立起非定域纠缠关联，以此实现多方代理人之间的联合秘密分发。

## 3.4 针对 QSS 的典型内鬼与不诚实攻击（高频简答分析题）

> 💡 **核心防御直觉**：秘密共享协议不仅要防外部窃听者 Eve，**更主要的是要防内部的“不诚实代理人”**。外部 Eve 偷听可以通过不确定性原理检测发现，而内部不诚实者可以利用手里的合法粒子流玩弄阴谋。

### 3.4.1 不诚实代理人的“假 Bell 态替换攻击”
- **攻击场景（以四方 ZM 协议为例）**：假设 Alice 是分发者，Bob、Charlie、David 是代理人。其中 **Bob 是不诚实的（内鬼）**。
- **攻击手段**：
  1. 在接收或转发粒子阶段，Bob 拦截了本来应该传给后续诚实代理人的粒子。
  2. Bob 自己偷偷制备一个完全受他控制的**伪造 Bell 态粒子对**，把其中一个粒子顶替塞入序列发给 Charlie，另一个自己留着。
  3. 随后，Bob 利用自己同时把守前后通道的优势，在后处理公开阶段不暴露身份，等诚实方宣布测量基后，Bob 在自己的小实验室里对留存的粒子做联合 Bell 基测量。
- **致命后果**：分析表明，如果首尾两个代理人（或者内部特定不诚实方）暗中勾结合作，**不诚实代理人可以在完全不需要其他诚实代理人帮助的情况下，绕过门限限制，提前独自截获重构出 Alice 的全部秘密信息**，而且在控制模式检测下错误率为 0。
- **防范对策**：在设计 QSS 协议时，各方粒子传递过程中必须加入无规律的、不可预测的**控制模式 (C-mode) 交叉混入抽取检查**，强行阻断内鬼在固定位置实施假态替换的时机。
