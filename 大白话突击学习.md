# 导论：量子世界的“不讲理”规则

经典密码学（如 RSA、AES）的安全性基于**数学难题**（算得慢，比如大数分解）；而量子密码学的安全性基于**物理定律**（做不到，比如你只要看一眼，答案就变了）。

要弄懂量子密码，必须先接受它最根本的四大核心底层直觉：

## 1. 量子态与叠加：旋转的硬币
在宏观世界里，一个硬币要么是正面（记为 $0$），要么是反面（记为 $1$）。
但在量子世界里，一个量子比特（Qubit）在没有被看之前，它是一枚**正在旋转的硬币**。

### 1.1 数学怎么描述这枚“旋转的硬币”？
我们用 [[狄拉克符号]] 把它写成：
$$|\psi\rangle = \alpha|0\rangle + \beta|1\rangle$$
- $|0\rangle$ 和 $|1\rangle$ 是两面（专业术语叫 **基向量** 或 **计算基**）。
- $\alpha$ 和 $\beta$ 是复数，代表它偏向哪一面的“分量”。
- **归一化条件**：$|\alpha|^2 + |\beta|^2 = 1$。这有什么物理意义？物理本质就是：这枚硬币落地后，是正面或反面的**概率总和必须是 100%**。

### 1.2 考试常考的两套“视角”（基）
量子密码里最常用的两个视角：
1. **$Z$ 基（十字基）**：以 `{|0⟩, |1⟩}` 分别代表上下、左右垂直极化。
2. **$X$ 基（对角基）**：以 `{|+⟩, |-⟩}` 分别代表 $45^\circ$ 和 $135^\circ$ 斜着极化。

它们之间可以相互翻译，就像把坐标轴旋转了 $45^\circ$：
- $|+\rangle = \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)$
- $|-\rangle = \frac{1}{\sqrt{2}}(|0\rangle - |1\rangle)$

**秒杀直觉**：只要记住 $\frac{1}{\sqrt{2}}$ 它的平方是 $\frac{1}{2}$（即 50%）。这意味着，用 $X$ 基的视角去看 $Z$ 基的状态，结果一定是完全随机的（各占 50%）。

# 2. 量子测量：残忍的塌缩与不可区分性

这是全课最核心的考点！**“看一眼”这个动作在量子力学里是有破坏性的。**

### 2.1 什么是测量（塌缩）？
测量不是“悄悄偷看”，测量是**用一堵带有特定栅栏的墙去挡住量子**。

如果你用 **$Z$ 基栅栏** 去测量叠加态 $|\psi\rangle = \alpha|0\rangle + \beta|1\rangle$：
- 这枚硬币会**瞬间强行停止旋转**（术语叫 **塌缩**）。
- 它有 $|\alpha|^2$ 的概率变成完完全全的 $|0\rangle$；
- 它有 $|\beta|^2$ 的概率变成完完全全的 $|1\rangle$。
- **一旦塌缩，原本旋转时的精妙细节（相位信息）全部灰飞烟灭！**



### 2.2 为什么“非正交态不可完美区分”？
- **正交态**：就像 $|0\rangle$ 和 $|1\rangle$，夹角 $90^\circ$。你用 $Z$ 基去测，如果它是 $|0\rangle$，测出来 100% 是 0；如果它是 $|1\rangle$，测出来 100% 是 1。这就叫可完美区分。
- **非正交态**：就像 $|0\rangle$ 和 $|+\rangle$，夹角是 $45^\circ$。
  - 如果你用 $Z$ 基去测 $|0\rangle$，能得到 0。
  - 如果你用 $Z$ 基去测 $|+\rangle$，它有 50% 的概率测出 0，50% 的概率测出 1。
  - **致命漏洞**：当你测出一个结果是 `0` 的时候，你**根本无法断定**原本Alice发过来的是 $|0\rangle$ 还是 $|+\rangle$！

> **期末核心结论**：想要百分之百区分非正交态，逻辑上要求必须在不破坏它的情况下探测它。但根据量子力学，**只要区分非正交态，就必然会干扰至少其中一个态**。这就是 BB84 协议安全的最高物理圣经。

# 3. BB84 协议的博弈：为什么窃听者必暴露？

掌握了上面的测量原理，我们直接拆解 BB84 协议的本质。

### 3.1 正常流程（Alice 和 Bob）
1. Alice 随机生成一串比特（比如 `0`），再随机选一个基（比如 $X$ 基），制备成量子态 $|+\rangle$ 发出去。
2. Bob 收到后，由于不知道 Alice 用了什么基，他就**猜一个**。
   - 猜对了（也选 $X$ 基）：100% 测出 `0`。
   - 猜错了（选了 $Z$ 基）：50% 概率出 `0`，50% 概率出 `1`。
3. **基比对（Sifting）**：双方在经典信道（比如打电话）里公开各自用的**基**（但不公开具体比特）。基一样的留下，基不同的扔掉。

### 3.2 截获-重发攻击（Eve窃听）：为什么她会引入 25% 的错误率？
窃听者 Eve 潜伏在信道中间，她想知道密钥。因为不可克隆，她不能复制一份，她只能**强行拦截并自己测量**。



我们来算一笔账（期末经典推导题）：
1. 假设 Alice 发送 $|0\rangle$ （$Z$ 基）。
2. Eve 拦截，由于她不知道基，她有一半的概率选错基（选了 $X$ 基去测 $|0\rangle$）。
3. 当 Eve 选错基时，她测出 $|+\rangle$ 或 $|-\rangle$ 的概率各是 $50\%$。假设她测出了 $|+\rangle$，量子态发生塌缩。Eve 只能按照她的测量结果重新发射一个 $|+\rangle$ 给 Bob。
4. Bob 此时收到的是已经被 Eve 污染过的 $|+\rangle$。
5. 在后处理中，Alice 和 Bob 比对时，只会留下大家都用 $Z$ 基的轮次。此时 Bob 用 $Z$ 基去测这个受污染的 $|+\rangle$，有 **$50\%$ 的概率测出错误答案 `1`**。

$$\text{总错误率} = \underbrace{50\%}_{\text{Eve 选错基的概率}} \times \underbrace{50\%}_{\text{Bob 在错态下测出错比特的概率}} = 25\%$$

**直觉记忆**：只要 Eve 动了手，就会在原本完美对称的相同基轮次中留下 **25% 的硬伤害错误率**。Alice 和 Bob 抽样检查发现错误率超标（比如大于 11% 理论硬阈值），就知道“有人在偷看”，直接扔掉密钥。

# 4. 量子纠缠：跨越时空的“超自然”同步

如果说单比特是硬币，[[量子纠缠]] 就是两个**无论相隔多远，只要一个落地朝上，另一个绝对瞬间朝下**的魔法硬币。

### 4.1 四个 Bell 态（最大纠缠态）的解构
考试最喜欢考这四个态的推导，请务必建立这种直觉：
- $|\Phi^+\rangle = \frac{1}{\sqrt{2}}(|00\rangle + |11\rangle)$ ：**同向同性**。左边测出0，右边必然是0；左边测出1，右边必然是1。
- $|\Psi^+\rangle = \frac{1}{\sqrt{2}}(|01\rangle + |10\rangle)$ ：**异向同性**。左边测出0，右边必然是1；左边测出1，右边必然是0。

### 4.2 为什么纠缠不能实现“超光速通信”？
*大题高频辨析考点*：既然纠缠态的塌缩是瞬间的，那我能不能利用它实现超光速发短信？
**答案：绝对不能。**

**人话解释**：
- Alice 测量她手里的粒子，结果是完全随机的（50% 概率出 0）。虽然 Bob 的粒子瞬间也固定了，但 Bob 没去测之前，他看到的依然是一枚旋转的硬币。
- Bob 必须去测，测完发现是个 1。但他怎么知道这个 1 是因为 Alice 测量导致的，还是自己运气好测出来的？
- **Alice 必须通过打电话（经典信道，受光速限制）告诉 Bob**：“我刚才测了，我这边是 0！”Bob 才能恍然大悟。所以，传递**有效信息**的速度依然没能超越光速。

# 5. 攻克大题：高级安全模型（DI / MDI / QSDC）

到了期末考试后半段，卷子开始拔高，通常会考下面三种方案的对比：

| 协议方案 | 核心对治的痛点 | 它的“安全潜台词” |
| :--- | :--- | :--- |
| **诱骗态方法 (Decoy)** | 工业光源不完美（偶尔发两个相同光子，会被 Eve 搞 [[PNS攻击|光子数分离攻击]]） | Alice 故意混入一些强弱不同的“诱骗光子”，Eve 只要敢拦截，不同光子的响应率就会失衡，直接暴露。 |
| **[[MDI-QKD|测量设备无关 (MDI)]]** | 工业探测器不完美（容易被 Eve 用强激光“致盲”或控制） | “最容易坏、最不安全的事情让外人去做。”Alice 和 Bob 只管发光，把测量设备扔给第三方（中继）。**只要中继报出的结果符合 Bell 基对应关系**，密钥就是安全的。彻底免疫了所有单光子探测器侧信道攻击。 |
| **[[DI-QKD|设备无关 (DI)]]** | 整个量子设备都是厂商造的，可能留了后门 | “我谁都不信，我把设备当黑盒。”直接拿数据去测 **Bell 不等式（CHSH 违背）**。只要违背值大于 2，哪怕设备是 Eve 造的，物理学定律也强行保证它没泄密。 |
| **[[QSDC|量子安全直接通信]]** | QKD 还要拿随机数去加解密，太麻烦 | Alice 直接把秘密信息编码在自己对纠缠粒子的局域操作上。利用**控制模式**和**消息模式**分步走，确保“Eve 没偷看，我再把最后一部分解密线索放过去”。 |
