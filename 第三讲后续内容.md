# 🎯 板块一：QKD 协议的安全性分析与实际漏洞（侧信道攻击）
在理论上，QKD 证明了可以达到**信息论无条件安全（无条件安全性证明）**。但在实际工程落地时，元器件并不像数学公式那样完美，这就给窃听者 Eve 留下了可乘之机。考试极易考查以下几种实际攻击手段的原理：

### 1. 特洛伊木马攻击（Trojan Horse Attack）

- **攻击原理：** Eve 并不在传输中截获光子，而是从外部**主动向 Bob（或 Alice）的系统内部发射一束强激光**。这束激光进入系统后，会经过系统内部的调制器（比如用来选择基的相位调制器）并被反射回来。Eve 截获这束反射回来的强激光，通过分析其相位或偏振，就能直接偷窥到 Alice 或 Bob 选了什么基。
- **防御手段：** 在入口处加装光隔离器（Isolator）和内置窄带滤波片，只允许单向光流出，阻止外来强光原路返回。

### 2. 探测器死时间与时域攻击（Timing Attack）

- **攻击原理：** 实际的单光子探测器在成功检测到一个光子后，需要一段恢复时间（称为**死时间**，Dead Time）。在这段时间内，探测器是“瞎”的。Eve 可以利用精确的时间控制，故意在特定时间发射光子，使得 Bob 的某个特定基的探测器陷入死时间状态，从而操控 Bob 的有效计数。
- **防御手段：** 采用随机延迟发射和硬件参数的动态校准。

### 3. 终极防御：测量设备无关 QKD（MDI-QKD）

- **核心痛点：** 绝大多数实际侧信道攻击都发生在**探测器端（Bob 端）**。
- **黑科技原理：** 既然探测器容易被攻击，那 Alice 和 Bob 都不负责测量。他们把光子同时发给中间一个**完全不可信的第三方 Charlie**，让 Charlie 做联合贝尔测量（Bell State Measurement）并公布结果。
- **考试结论：** 哪怕 Charlie 本身就是间谍 Eve 伪装的，他也只能知道 Alice 和 Bob 的相对态关系，而无法获知密钥的绝对值。MDI-QKD 完美堵死了**所有针对探测器的侧信道攻击**，是现代量子密码工程的核心突破。

# 🎯 板块二：密钥分配的后处理阶段（Post-Processing）
光子在光纤里跑，即使没有 Eve 窃听，环境噪声（光纤双折射、热噪声）也会导致 Bob 测错，产生**固有误码率**。所以筛选密钥（Raw Key）还不能直接当密码用。必须经过以下四个经典信息处理步骤（**常考流程填空或简答**）：

```
[筛选密钥 Raw Key] 
       │
       ▼
 1. 纠错 (Information Reconciliation) ──> 消除信道噪声导致的错位
       │
       ▼
 2. 隐私放大 (Privacy Amplification)  ──> 压缩 Eve 可能窃听到的残余信息
       │
       ▼
 3. 密钥校验 (Authentication)         ──> 确认两端密钥完全一致
       │
       ▼
[最终安全密钥 Final Key]

```

1. **基比对（Sifting）：** 抛弃基不匹配的比特，留下筛选密钥。
2. **纠错（Information Reconciliation）：** Alice 和 Bob 通过经典信道交换少量的校验信息（通常采用 Cascade 协议或高效率的 **LDPC 码**），把噪声引起的零星错比特给纠正过来，使两端密钥 100% 相同。
3. **隐私放大（Privacy Amplification）：** 这是安全性的关键！即使错误率很低，Eve 也有可能通过偷听获取了极其微量的残余信息。Alice 和 Bob 在本地使用 **Hash 函数（散列函数）** 对纠错后的密钥进行非线性压缩。
  - **数学效果：** 将一串 Eve 知道一点点的长密钥，压缩成一串稍短、但 Eve 的知晓度呈**指数级衰减至几乎为零**的纯净最终密钥。
4. **身份认证（Authentication）：** 防止 Eve 在经典信道进行“中间人攻击”，确保和自己通话的确实是对方。

# 🎯 板块三：多维度与连续变量 QKD（扩展视野考点）
除了最经典的离散变量（DV-QKD，如用光子的偏振、相位表示 0 和 1），第三讲后半部分通常还会提及另一种技术分支：

- **连续变量 QKD（CV-QKD）：**
  - **物理本质：** 它不使用昂贵、脆弱的单光子探测器，而是利用光场的**正交振幅**和**正交相位**（连续变量）来编码信息。
  - **优点（常考选择题）：** 可以直接利用现有的、非常成熟的**经典相干光通信器件（如平衡零点探测器）**，在城市骨干网中更容易低成本部署。
  - **缺点：** 对相位噪声极其敏感，长距离传输时信号衰减严重，目前传输距离普遍低于离散变量协议。
